# Database Schema Design

TransitOps uses **PostgreSQL** configured via **node-postgres (pg)** connection pooling. There is **no ORM (like Prisma)** to ensure maximum query performance, auditability, and ease of database management directly inside pgAdmin 4.

## 📁 Database Directory Layout

All raw database files are stored in the monorepo root:

```
database/
├── schema.sql         # Base DDL table structures
├── migrations/        # SQL migration history
│   └── 001_initial_schema.sql
├── seeds/             # Seed SQL scripts (default roles, permissions)
│   └── seed.sql
├── indexes/           # Performance index DDL definitions
│   └── indexes.sql
├── functions/         # Custom triggers and functions
│   └── functions.sql
└── views/             # Analytical views
    └── views.sql
```

## 🛠 Database Schema Conventions

- **Naming:** Lowercase snake_case table names (`users`, `user_roles`, `fuel_logs`).
- **Keys:** Every primary key is a UUIDv4 (`id`) generated by the `uuid-ossp` extension.
- **Soft Delete:** A `deleted_at` timestamp field indicates logical deletion. Queries should always filter by `deleted_at IS NULL` unless fetching history.
- **Audit Columns:**
  - `created_at` (timestamp with time zone, defaults to current time)
  - `updated_at` (timestamp with time zone, defaults to current time)
  - `created_by` (optional user UUID)
  - `updated_by` (optional user UUID)

## 📋 Table Indexing Strategy

To maintain high performance as data scales, the following indexes are generated:

- `users(email)`
- `roles(code)`
- `permissions(code)`
- `vehicles(plate_number)`
- `trips(vehicle_id, driver_id, status)`
- `maintenance(vehicle_id, status)`
- `fuel_logs(vehicle_id)`
- `expenses(expense_category_id)`
- `notifications(user_id, is_read)`
- `activity_logs(user_id)`
- `audit_logs(entity_name, entity_id)`

Additionally, a composite unique index is set on mapping tables `user_roles(user_id, role_id)` and `role_permissions(role_id, permission_id)`.
